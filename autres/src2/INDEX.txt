🔑 KEY ARCHITECTURAL CHANGES
═════════════════════════════════════════════════════════════════════════════

1. CLEAR SEPARATION OF CONCERNS
   AppState    (persistent)  ←→  AppRun (runtime)  ←→  NumericContext (compute)
   
   ✓ AppState: Only serializable fields (dark, panel, layout, controller)
   ✓ AppRun: Only runtime objects (Observable, Task, Channel, Atomic)
   ✓ NumericContext: Immutable compute data (IRF, FFT plans)

2. NO GLOBAL STATE MUTATIONS
   OLD: global irf = ...; global fft_plan = ...
   NEW: num_context = NumericContext(...)  [passed explicitly]
   
   ✓ Thread-safe by design
   ✓ No race conditions
   ✓ Testable (no hidden dependencies)

3. TYPED CONFIGURATION (not Dict)
   OLD: app.layout[:time_range]  [returns Any]
   NEW: app.layout.time_range    [returns Int]
   
   ✓ Compiler validates field access
   ✓ IDE autocomplete works
   ✓ ~2x faster numeric code

4. ATOMIC PERSISTENCE
   OLD: open(path, "w") { serialize } [crashes mid-write → corruption]
   NEW: write(.tmp) → mv(.tmp, path)  [atomic at filesystem level]
   
   ✓ Crash-safe
   ✓ Version tag for future migrations
   ✓ Load handles old formats

5. PROPER TASK LIFECYCLE
   OLD: @async { wait(task) }  [returns immediately, no cleanup]
   NEW: close(channel); wait(tasks...)  [synchronous, complete]
   
   ✓ No resource leaks
   ✓ Safe window close
   ✓ Predictable shutdown

Q: Can I keep src/ and src2/ side by side?
A: Yes! They're independent. Use symlinks or update include paths.

Q: What if I have custom modifications in src/?
A: Review CHANGES.md and manually merge changes into corresponding files.

Q: Will old .jls state files still load?
A: Yes! load_state() has fallback for unversioned format.

Q: Do I need to change Project.toml?
A: Only if dependencies changed (unlikely; src2 uses same libs).

Q: How do I run with multiple Julia threads?
A: julia --threads 4 src/main.jl

Q: What if NumericContext is too large?
A: It's not (3 numbers + 2 FFT plans = ~1MB). Pass by reference fine.

🎓 LEARNING RESOURCES
═════════════════════════════════════════════════════════════════════════════

Julia Best Practices (in code):
- Type-stable functions (no Union in return types)
- Immutable structs for data (easier reasoning)
- Pass context objects instead of globals
- Use @async sparingly; wait() for coordination
- Observable = reactive value (assignment triggers notify)

Domain Knowledge (FLIM):
- IRF = system impulse response (measured once per setup)
- MLE = maximum likelihood estimation (optimization of likelihood)
- Convolve = model(x) ⊗ IRF + noise
- Lifetime = exp decay time constant (what we fit)

Makie (GUI):
- GridLayout = regular grid of UI components
- Observable{T} = reactive container (change → notify listeners)
- @async = spawn green thread (Task)
- Channel{T} = FIFO queue for inter-task communication
