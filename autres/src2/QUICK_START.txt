DataTypes.jl
────────────
Defines core data structures:
- AppState: Persistent GUI state (can be serialized)
- AppRun: Runtime state (Observables, Channels, Tasks)
- NumericContext: Numeric data (IRF, FFT plans)
- LayoutState: Typed layout configuration
- ControllerState: Typed controller PID settings
+ Factory functions: create_app_run(), create_default_app_state()


Persistence.jl
──────────────
State save/load with versioning:
- save_state_atomic(): Atomic write + versioning tag
- load_state(): Loads with migration support
- ensure_state_dir(): Creates docs/ if needed
✓ Crash-safe pattern: write .tmp → atomic move → success


Attributes.jl
─────────────
Makie GUI theming:
- Color palettes (dark/light modes)
- Helper functions: get_theme_colors(), update_colors_for_theme()
- UI component attributes: BUTTON_ATTRS, LABEL_ATTRS, etc.
✓ Reduced boilerplate vs src/


main.jl
───────
Application initialization & lifecycle:
1. Load/create AppState
2. Create AppRun runtime state
3. Load IRF and initialize FFT plans → NumericContext
4. Create GUI
5. cleanup_app_run() on exit ← PROPER SHUTDOWN


functions.jl
────────────
File I/O and worker tasks:
- list_ports(): Enumerate serial ports (OS-aware)
- open_SDT_file(): Read Becker & Hickl .sdt binary format
- worker_test(): Test data worker (reads .sdt, fits lifetime, sends via channel)
  * Renamed from test() for clarity
  * Accepts num_context instead of globals
✓ Docstrings with references


vec_to_lifetime.jl
──────────────────
Fluorescence lifetime extraction via MLE:
- get_irf(): Load IRF from .sdt file
- get_irf_bin_size(): Compute time resolution
- convolve(): FFT-based IRF convolution
- irf_shift(): Apply instrumental delay correction
- conv_irf_data(): 4 overloads for (1,2,3,4)-exponential models
- MLE_iterative_reconvolution_jl(): Curve fitting optimizer
- vec_to_lifetime(): Main public API
✓ Accepts NumericContext (no globals!)
✓ Math formulas in docstrings
✓ Error handling with logging


GUI.jl
──────
Makie GUI creation and task management:
- make_gui(): Creates figure with all plots and controls
- start_consumer!(): Channel reader → Observable updater
  * FIXED: Uses vcat() assignment for Observable notify
- start_autoscaler!(): Periodic plot limit updates
- start_infos!(): Live frequency counter
- Plot mapping: histogram, photon counts, lifetime, concentration, command
✓ Proper task lifecycle
✓ Correct Observable updates


handlers.jl
───────────
Panel event handlers:

# 🔄 MIGRATION FROM src/ TO src2/
# ============================================================================

Main BREAKING CHANGES:
– app.layout is now a struct, not Dict
  OLD: app.layout[:time_range]
  NEW: app.layout.time_range

– appRun.channel is no longer global "irf"
  OLD: global irf = get_irf()
  NEW: num_context = NumericContext(...); vec_to_lifetime(x, num_context; ...)

– Task shutdown is different
  OLD: @async try; wait(task); catch ...
  NEW: call cleanup_app_run!(app_run) cleanly

– Observable updates changed
  OLD: push!(app_run.photons[], photons)
  NEW: app_run.photons[] = vcat(app_run.photons[], [photons])

See README.md and CHANGES.md for detailed before/after examples.

# ============================================================================
# 🐛 TROUBLESHOOTING
# ============================================================================

ERROR: IRF file dialog never opens
→ Check if NativeFileDialog.jl is installed
→ Try manually editing docs/irf_filepath.txt with absolute path

ERROR: Channel closes unexpectedly
→ Check worker_test() for exceptions
→ Look for "Mismatch" warnings (histogram resolution issue)

ERROR: Plots don't update after START
→ Verify Observable assignment using vcat() (not push!)
→ Check consumer_task is running (@async begin...end)
→ See if channel is being read (look at put!(ch, ...) in worker_test)

ERROR: STOP button hangs
→ Task is not exiting (check running[] flag in worker_test)
→ Channel is not being closed before wait()
→ See cleanup_app_run!() in main.jl

ERROR: StateFile corrupted
→ Delete docs/AppState.jls and docs/AppState.jls.tmp
→ Restart app to regenerate with defaults
